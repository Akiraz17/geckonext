# Gecko Next — Рабочая среда разметки аудио/видео реплик

Современное веб-приложение на React + TypeScript для точной сегментации звуковых дорожек, редактирования реплик спикеров, разметки кросстоков и экспорта таймкодов в стандартизированный формат Gecko JSON.

## ✨ Основной функционал

- 📊 **Интерактивный плеер**: Визуализация звуковой волны через `wavesurfer.js` с управлением масштабом.
- ↩️ **Умная отмена действий (Undo)**: История изменений сегментов с поддержкой клавиатурного хоткея `Ctrl + Z`.
- ✂️ **Быстрое разделение**: Разрезание монологов по метке текущего воспроизведения с интеллектуальным разделением слов (`terms`).
- 🌗 **Стабильный темный режим**: Полная поддержка темной и светлой темы с защитой инпутов от сброса цветов Tailwind.

Для запуска вам понадобятся установленные **Node.js** (v18+) и **npm**.

git clone [https://github.com/ВАШ_НИКНЕЙМ/gecko-next.git](https://github.com/ВАШ_НИКНЕЙМ/gecko-next.git)

## 🚀 Быстрый запуск на любом ПК

Вам нужен только установленный **Node.js**. Больше ничего настраивать вручную не придётся:

- **На Windows:** Просто дважды кликните по файлу `install-and-run.bat` в корне проекта.
- **На macOS / Linux:** Откройте терминал в папке проекта и выполните `./install-and-run.sh`.

Скрипт сам проверит окружение, докачает нужные библиотеки и запустит локальный веб-сервер.

## Установка руками

- Скачайте исходный код проекта с удаленного репозитория GitHub и перейдите в его корневую директорию:
git clone [https://github.com/ВАШ_НИКНЕЙМ/gecko-next.git](https://github.com/ВАШ_НИКНЕЙМ/gecko-next.git)
cd gecko-next
- Поскольку проект спроектирован в виде изолированного клиентского приложения, перейдите в целевую подпапку frontend, где находятся файлы package.json и vite.config.ts:
cd frontend
- Запустите процесс скачивания пакетов React, Tailwind CSS, WaveSurfer.js и компилятора TypeScript:
npm install
- Поднимите сервер разработки сборщика Vite с активной функцией Hot Module Replacement (HMR, автообновление интерфейса при сохранении файлов кода):
npm run dev
- По завершении сборки в окне терминала отобразятся доступные локальные адреса. Скопируйте и откройте в браузере стандартную ссылку.

- 📁 **Спецификация формата импорта и экспорта данных**
- Приложение спроектировано под взаимодействие со структурированным стандартом файлов Gecko JSON (v2.0). Всплывающее модальное окно при старте потребует от вас загрузки исходного медиафайла (видео или аудио) и метаданных .json следующего формата:
{
  "schemaVersion": "2.0",
  "monologues": [
    {
      "speaker": { "id": "SPEAKER_00", "name": "SPEAKER 00" },
      "start": 0.01,
      "end": 1.69,
      "text": "Привет, мир.",
      "crosstalk": false,
      "terms": [
        { "text": "Привет,", "type": "WORD", "start": 0.01, "end": 0.93 },
        { "text": "мир.", "type": "WORD", "start": 0.95, "end": 1.69 }
      ]
    }
  ]
}
